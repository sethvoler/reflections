初始化版本库时会发生什么情况？为什么你需要这样做？
    git init初始化版本库，git status确保这是一个git代码库。

暂存区与工作目录和版本库有何不同？你认为它具有什么价值？
    暂存区是一个工作目录到版本库的过渡区域。可以作为一个缓存区域，然后考虑是否真的放入版本库。

如何使用暂存区确保为每项逻辑更改进行一次提交？
    git add将文件放入暂存区，git commit写入commit消息

在哪些情况下，分支可以帮助你有序地组织历史记录？分支将如何提供帮助？
    创建不同版本或者进行实验性的代码编写时。分支不会导致主干线上的代码混乱。

图表如何帮助你直观显示分支结构？
    图表比较形象。

将两个分支合并到一起有何结果？为什么我们在图表中表示它的方式是那样的？
    两个分支的提交将合并到一起。因为合并并不会自动删除合并的分支。

Git 的自动合并与始终手动进行合并各有何优缺点？
    自动合并比较快，但是发生冲突就无法解决；手动合并比较慢，但是可以解决冲突.
